LESSON 2

Where can you place Scripts?
There are times when we want to execute the codes depending on the situation. It could be pre-loaded, or to be triggered whenever an event happens. This is possible in JavaScript. You can control the execution of the codes by placing it in different sections of the HTML file.

HEAD
Whenever the scripts are placed here, you are assured that they will be pre-loaded – it means the scripts will be executed before anyone triggers an event. Scripts are ideally placed here are for functions calls.
BODY
Scripts that are placed here are executed when the page loads.
HEAD & BODY
You can place your script tags whenever and how many you want to.

EXTERNAL JAVASCRIPT
External JavaScript files are helpful especially if you are using them to control different HTML files. There is no need to rewrite them all over again. Save the file having an extension of .js. To use the JavaScript file, all you have to do is use the src attribute and provide the destination of the file.
The src attribute will point to the source of the JavaScript external file. Take note that the JavaScript file does not contain the opening and closing script tags.
There are times when you want to add a statement to explain a set of codes, or when we want to omit certain codes for debugging purposes, this can be done by inserting a comment. For single comments, you can use two backlash symbols; for multiple line or block comments, start with a /* and end with a */.

What are the JavaScript Guidelines?
In coding JS, it is important to know the Do’s and Don’ts. This will save you time and keep you away from trouble in case errors occur. Listed below are the guidelines.

Case Sensitive
You must be careful in naming your variables. The variable myHome is not equal to MyHome. It is important to keep track of your variables so there won’t be problems in calling variables, objects or functions. Unlike in HTML, uppercase letters are not equivalent to lowercase letters in JavaScript.
White Space
JavaScript ignores white space as long as it is within the line of the code. It is a good practice to use white spaces between expressions to make your codes easier to read
Breaking up of code lines
You can break a code line by using a backslash (‘\’). However, do not break the code in between commands or else it will not work properly.

Example:
document.write(“wow\Philippines”); - Accepted
document.\write(“wow Philippines”) - Not accepted

LESSON 3

    Characteristics of JavaScript
  1. Interpreted Language
    JavaScript is an interpreted language, meaning it needs a web browser (like Chrome, Firefox, or Edge) to run. The browser reads and executes the code directly instead of compiling it first. JavaScript works together with HTML to make webpages interactive. The source code can easily be viewed when someone checks the webpage’s source.
  2. Scripting Language
    JavaScript is a scripting language, which means it is used to automate simple or repetitive tasks in a program. It does not control the entire structure of the webpage but instead adds functionality, such as changing elements, validating forms, or reacting to user actions.
  3. Client-Side Scripting
    There are two types of web scripting:
    • Client-side scripting – runs on the user’s computer/browser (JavaScript).
    • Server-side scripting – runs on the server hosting the webpage.
    • JavaScript is mainly client-side, meaning the script is executed by the user's browser.
    • Object-Oriented Programming (OOP)
      JavaScript supports Object-Oriented Programming, which means many elements are treated as objects. Each object has:
      • Properties – characteristics or attributes of the object.
      • Methods – actions the object can perform.

Key Concepts for Understanding JavaScript

Objects
items that exist in the browser environment such as:

  • browser window
  • webpage
    • status bar
    • date and time

Methods
functions associated with objects that perform actions, such as calculations or displaying content.

Properties
describe the characteristics of an object.

Event
an action that occurs on a webpage, such as:

  • clicking a button
  • loading a page
  • moving the mouse over a link
Scripts can be triggered when these events occur.

Parameter
data passed into a function or method to describe or modify an object.

Type
refers to the kind of data stored in a variable (such as number, string, or boolean).

Type Conversion
changing a variable from one data type to another.

Script
a set of computer instructions that combines different components to perform a task. JavaScript scripts are used to create interactive features on websites.

To write JavaScript, you usually need a text editor and a web browser; the basic components of JavaScript are objects, methods, and properties.

Common JavaScript Objects

Window Object– the main object in JavaScript and represents the browser window.

    Examples:
  • location – moves to another webpage
  • open() – opens a new window
  • setTimeout() – runs an event after a time delay
Usually, you do not need to write `window.` because it is assumed automatically.

Document Object– represents the webpage itself and contains all HTML elements.

    Examples:
  • write() – displays text on the webpage
  • lastModified – shows the last date the document was modified
  • fgColor – sets the foreground color
  • cookie – reads cookie information

Math Object– allows JavaScript to perform mathematical calculations.

    Examples:
  • PI – value of π
  • pow(a,b) – raises a number to a power
  • max(a,b) – returns the larger number
  • min(a,b) – returns the smaller number

Navigator Object– identifies information about the user's browser.

    Examples:
  • appName – browser name
  • appVersion – browser version
  • cookieEnabled – checks if cookies are enabled
  • platform – operating system description

Structure of a JavaScript Code Snippet
JavaScript is usually placed inside the section of an HTML document.
The <script> tag tells the browser that the content is JavaScript.
The browser parses the code, meaning it analyzes the text and converts it into instructions it can execute.

Variables– a container that stores data which can change during program execution.
Variables declared inside a function can only be used within that function.

    Data Types in JavaScript
    Variables can store different types of data:
  • Character – single letter or symbol
  • String – group of characters
  • Integer – whole numbers
  • Float / Double – decimal numbers
  • Boolean – true or false values
  • Function – block of instructions
  • Object – browser-related items
  • Array – collection of values
  • Undefined – variable without value
  • Null – variable intentionally set to empty

Rules for Naming Variables
Variable names must follow these rules:

  • Can contain letters, numbers, and underscores.
  • Cannot contain spaces.
  • Cannot use punctuation marks.
  • Must not start with a number.
  • Are case-sensitive. (firstname & FirstName are different)

LESSON 4

JavaScript Events – actions that trigger JavaScript code.

    Examples include:
  • mouse clicks
  • keyboard presses
  • page loading
  • submitting a form
Events usually work together with functions, which contain the code that will run when the event happens.

Common Event Triggers

  1. onload and onunload
    • These events occur when a webpage opens or closes.
    • They can be used to:
      • check browser information
      • load or save cookies
  2. onfocus, onblur, onchange
    • These are commonly used in form validation.
      • onfocus – when a field becomes active
      • onblur – when the field loses focus
      • onchange – when the value of a field changes
  3. onsubmit
    • This event is used when a form is submitted.
      Example: onsubmit=checkFields()
      The function checks if the form fields are valid before submitting.
  4. onmouseover and onmouseout
    • These events happen when the mouse pointer moves over or away from an element.
    • They are often used for animations or hover effects.
  5. onclick
    • Triggered when a user clicks an element, such as a button or link. It can be used to open webpages or trigger functions.

Popup Boxes – small windows that appear on the screen. JavaScript has three main types:

  1. Alert Box
    • Displays a warning or message
    • Syntax:alert("Message");
  2. Prompt Box
    • Requests input from the user.
    • Syntax: prompt("message","default value");
    • If the user presses OK, the value is returned.
    • If Cancel is pressed, the result is null.
  3. Confirm Box
    • Used to confirm a decision.
    • Syntax: confirm("message");
    • OK → returns true
    • Cancel → returns false
  4. Variables
    • temporarily store values or expressions.
    • Example:
      var x = 5;
      var y = x + 5;
      var fan = "Standard";
      var isDone = false;
    • JavaScript variables are loosely typed, meaning you do not need to specify the data type beforehand.
  5. Arithmetic Operators
    • JavaScript uses common mathematical operators:
      • + addition (can also combine strings, called concatenation.)
      • - subtraction
      • * multiplication
      • / division
      • % modulus
  6. Order of Operations
    • JavaScript follows this order:
      1. Parentheses
      2. Multiplication, division, modulus
      3. Addition and subtraction
  7. Increment and Decrement Operators increase or decrease a value by 1.
    • x++
    • ++x
    • x--
    • --x
      • Their result may differ depending on whether the operator is used before or after the variable.
  8. Comparison Statements check relationships between values.
    • Examples include:
      • == equal
      • != not equal
      • > greater than
      • < less than
  9. Logical Operators evaluate multiple conditions.
    Examples:
    • AND (&&) – both conditions must be true
    • OR (||) – at least one condition must be true
    • NOT (!) – reverses a condition
  10. Conditional Operator
    JavaScript has a shorthand condition called the ternary operator.
    Syntax: variable = (condition) ? value1 : value2;
    Example: myFavorite = (color == "pink") ? "Terrific" : "Change color";
    If the condition is true, `"Terrific"` is assigned. Otherwise, `"Change color"` is assigned.
    • Conditional Statements
      These statements control program flow.
    • if statement
      Executes code if the condition is true.
    • if–else statement
      Runs one block if true and another if false.
    • if–else if–else statement
      Allows multiple conditions to be checked.
    • switch statement
      Selects one option from many possible cases.
    • break statement
      Prevents the program from continuing to the next case.
